# 문서 정합성 QA (이격 검토)

문서를 작성/수정하면 **상위↔파생 문서 간 이격**을 대조한다. 실행: `Workflow({ name: "doc-consistency-qa", args: { service: "<svc|all>" } })`. 결과는 `docs/qa/doc-consistency-report.md`.

## 이격 매트릭스 (대조 쌍 → 기준)
```
설계(software) ─ DB(erd)        엔티티↔테이블, enum값 집합, 상태머신↔status 컬럼
DB(erd) ─ API(dto)             컬럼↔DTO 필드(명칭·타입·필수·마스킹)
설계 ─ API                      유스케이스/port↔엔드포인트·권한
설계 ─ 연동(integration)        이벤트분류·아웃박스↔메시지 스키마
DB/API ─ 연동                   페이로드 필드↔컬럼/DTO
API ─ PRD                       PRD 화면 호출 데이터↔엔드포인트·필드
PRD ─ PPT                       화면 요소(검색·컬럼·입력·버튼·탭·배지) 전수 대응 · 화면 순서·기능 ID 일치
PPT 흐름(내부)                   ① 탭 연속성: 멀티탭 상세/플로우 화면의 탭이 1탭=1슬라이드·같은 부모 탭 바·active 일치인지(여러 탭을 한 슬라이드에 욱여넣거나 빈 라벨 탭·부모와 다른 탭 바로 분리된 화면=흐름 단절). ② 드릴다운 진입 트리거: 다른 기능 ID로 가는 상세/드릴다운 화면 첫 슬라이드에 진입 배너(↩ 진입 경로)가 있고 소스 화면 행/버튼에 ▶+"→XXX"가 있는지(없으면=진입 경로 불명). ③ 변경이력 슬라이드 행 겹침(긴 문구) 여부. SKILL backoffice-planner §1.6·원칙 7 기준.
설계/DB/API/연동 ─ 서비스WBS      태스크(docs/tasks/<svc>) 참조 섹션 실재·범위 일치
서비스WBS ─ 프로그램로드맵         서비스 WBS T-ID ↔ 로드맵 Phase 태스크(docs/tasks/aegis-aml) 매핑·의존·범위·Status 정합
프로그램로드맵 ─ 설계/PRD          Phase 태스크 참조(설계 §·화면·엔드포인트·com.aegis 패키지) 실재
횡단(전 문서)                    멀티테넌시 키·PII 마스킹·규제·표시값↔enum 네이밍 일관
```

## 이격 판정·보고
- 심각도: **높음**(개발 직접 차질: 필드/타입/enum/엔드포인트 불일치·누락) / 중간(명칭·표시용어 불일치) / 낮음(설명 보강).
- 각 발견: 심각도·대조쌍·서비스·문서A/B·항목·이격내용·권고·위치.
- 정본 기준은 `_shared/target-architecture.md`. 높음 0건일 때 PASS, 아니면 FAIL.
- 상세 절차/검수 항목은 `design-reviewer` 에이전트가 수행한다.
