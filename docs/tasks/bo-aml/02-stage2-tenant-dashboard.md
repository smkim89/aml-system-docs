# BOA-S2 · 대시보드·고객사 관리 (AML-DASH-001 · AML-TNT-001/002/003)

> **목표**: AML 운영 콕핏(대시보드)과 고객사 생명주기(목록→상세 4탭→등록) + **보고기관 정보·소스 인입 신호** 완성.
> **선행**: BOA-S1. **PPT**: 슬라이드 3~9.

## 공통 참고 문서
- PRD §2.1(대시보드), §13.1~§13.3(고객사 — 배포 모델·온보딩), §1.8·§1.9(배포·온보딩 상태머신), §1.11(인입 신호 — ③ 탭 표준)
- API `02-aml-api.md` §9(bo-api 소유 `/api/v1/bo/aml/*`)·§3.16(TenantDto·온보딩)·§5(OpenAPI tenants)
- 표시: 배포 유형/온보딩 8종/운영 상태 4종 = PRD 부록 F

## DoD
- 대시보드 6카드·운영 알림 딥링크가 각 Stage 라우트로 연결. 기한 임박 보고 카드 = 법정 SLA(STR 3영업일·CTR 30일) D-3 규칙.
- 보고기관 정보 편집이 감사 기록되고, S5 REP-002 ① 헤더에서 파생 표시(통합 확인은 S5).
- ③ 탭 신호가 §1.11 확정 표준과 동일, [인입 모니터링]은 S6 ING-001로 연결(미구현 시 placeholder).
